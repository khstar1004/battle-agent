좋아. 데모의 목표는 **“실제 완성 제품”을 보여주는 게 아니라, 심사위원이 30초 안에 ‘아, 이건 단순 챗봇이 아니라 작전계획을 검증하는 AI 참모진이구나’라고 이해하게 만드는 것**이야.

# 최종 데모 Goal

## **작전계획서를 넣으면, AI 참모진이 계획을 깨뜨려보고, 지휘관 결심카드를 만들어낸다.**

데모는 이 한 문장을 증명하면 된다.

---

# 데모에서 반드시 보여줘야 하는 5가지

## 1. 작전계획 입력

심사위원이 처음 봤을 때 이런 화면이어야 해.

> “아, 실제 군에서 쓰는 계획서를 넣는구나.”

### 화면 요소

* 작전명
* 임무
* A/B/C 방책
* 병력·장비 현황
* 보급·정비 제한
* 통신 제한
* 기상 조건
* SOP / 지침 문서
* 적 예상 방책

### 목업 시나리오 추천

**대대급 야간 기동훈련 방책 검토**

| 항목   | 목업 데이터                             |
| ---- | ---------------------------------- |
| 작전명  | ○○대대 야간 기동훈련                       |
| A안   | 최단 경로 신속 전개                        |
| B안   | 우회 경로 안정 전개                        |
| C안   | 단계별 전개 후 예비대 대기                    |
| 제한사항 | 통신 음영구간 1개, 보급 대기점 부족, 차량 2대 정비 제한 |
| 기상   | 새벽 안개, 강수 가능성                      |
| 적 상황 | 지연행동 가능성, 우회로 감시 가능성               |
| 의무   | 후송 가능 경로 1개 제한                     |
| SOP  | 통신두절 30분 이상 시 재보고·재판단 필요           |

---

## 2. 단순 LLM 결과

이건 반드시 넣어야 해.
왜냐하면 심사위원이 “그냥 ChatGPT로 하면 되는 거 아닌가?”라고 생각할 수 있기 때문이야.

### 목표

**단순 LLM은 빠르지만 일반론적이라는 걸 보여준다.**

### 예시 출력

> A안은 신속성이 높고 B안은 안정성이 높습니다.
> 기상 악화와 차량 정비 상태를 확인하고, 통신대책과 안전교육을 강화해야 합니다.
> 종합적으로 A안을 우선 고려하되 위험요소를 보완하는 것이 적절합니다.

이렇게 일부러 평범하게 보여줘야 해.

### 화면 제목

**단순 LLM 분석 결과: 빠른 요약, 낮은 검증밀도**

---

## 3. 멀티에이전트 병렬토론 화면

이게 데모의 핵심이야.

### 목표

**AI 참모들이 각자 다른 관점으로 계획을 공격하는 장면을 보여준다.**

### 반드시 들어갈 에이전트

| 에이전트       | 데모에서 하는 말                          |
| ---------- | ---------------------------------- |
| 적군 에이전트    | A안은 최단 경로라 예측 가능성이 높고 지연행동에 취약     |
| 군수 에이전트    | A안은 보급 대기점 부족으로 작전 지속성 위험          |
| 통신 에이전트    | A안 경로 중간에 통신 음영구간 존재               |
| 기상·지형 에이전트 | 새벽 안개로 이동속도 저하 가능                  |
| 의무후송 에이전트  | 사고 발생 시 후송 경로가 제한됨                 |
| SOP 에이전트   | 통신두절 시 재판단 기준이 명령서에 없음             |
| 레드팀 에이전트   | 통신장애 + 보급지연 + 적 지연행동이 결합되면 지휘공백 발생 |
| 심판관 에이전트   | 근거 있는 주장만 결심카드에 반영                 |

### 화면 제목

**AI 참모진 병렬토론: 계획을 깨뜨려보는 과정**

---

## 4. 실패경로 도출 화면

이 화면은 보고서 캡처용으로 매우 중요해.

### 목표

**멀티에이전트가 ‘예상치 못한 결합위험’을 찾아냈다는 걸 보여준다.**

단순히 위험요소 나열이 아니라, 반드시 **결합된 실패경로**로 보여줘야 해.

### 예시

| 순위 | 실패경로                        | 영향        | 근거                  |
| -: | --------------------------- | --------- | ------------------- |
|  1 | 적 지연행동 + 통신 음영 + 예비대 투입 지연  | 지휘공백      | 적군·통신·작전 에이전트 공통 지적 |
|  2 | 보급 대기점 부족 + 차량 정비 제한        | 작전 지속성 저하 | 군수 에이전트 근거          |
|  3 | 새벽 안개 + 최단 경로 이동 + 후송 경로 제한 | 사고 대응 지연  | 기상·의무 에이전트 지적       |
|  4 | 통신두절 기준 미명시 + SOP 불일치       | 재판단 지연    | SOP 에이전트 지적         |

### 화면 제목

**주요 실패경로 TOP 4**

---

## 5. 지휘관 결심카드

최종 데모의 끝은 무조건 이 화면이어야 해.

### 목표

**지휘관이 실제로 쓸 수 있는 결과물로 끝낸다.**

긴 리포트 말고 1페이지 카드.

---

# 최종 출력 예시

## **BATTLE-PROOF K 지휘관 결심카드**

**종합판단**
B안 우선 권고. A안은 조건부 시행 가능.

**판단근거**
A안은 속도 면에서 유리하지만, 적 지연행동·통신 음영·보급 지연이 결합될 경우 지휘공백 위험이 높음.
B안은 시간이 더 걸리지만 작전 지속성과 후송 안정성이 높음.

| 항목       | 내용                                    |
| -------- | ------------------------------------- |
| 권고 방책    | B안 우선                                 |
| A안 시행 조건 | 예비 통신수단 지정, 보급 대기점 추가, 재판단 기준 명시      |
| 주요 리스크   | 지휘공백, 보급지연, 후송제한                      |
| 즉시 수정안   | 명령서에 재판단 기준 추가, 예비차량 지정, 의무후송 대체경로 지정 |
| 지휘관 결심사항 | A/B/C안 선택, 보급대기점 추가 승인, 예비통신수단 지정     |

# 데모에서 반드시 박아야 할 비교 지표

데모에 이 표 하나는 꼭 넣어야 해.

## 단일 LLM vs BATTLE-PROOF K

| 항목       | 단일 LLM | BATTLE-PROOF K |
| -------- | -----: | -------------: |
| 분석 시간    |     2분 |             5분 |
| 발견한 위험요소 |     4개 |            11개 |
| 결합 실패경로  |     1개 |             4개 |
| 근거 표시    |     일부 |         90% 이상 |
| 수정안 구체성  |    일반적 |        실행조치 단위 |
| 최종 산출물   |    요약문 |       지휘관 결심카드 |

핵심은 이거야.

> **단일 LLM은 위험요소를 나열한다.
> BATTLE-PROOF K는 실패경로를 찾아 수정안을 낸다.**

# 데모의 최종 성공 기준

데모를 본 사람이 아래 3개를 이해하면 성공이야.

1. **단순 LLM은 부족하다.**
   요약은 하지만 작전계획을 깊게 공격하지 못한다.

2. **멀티에이전트가 필요한 이유가 보인다.**
   적군, 군수, 통신, 기상, 의무 관점이 충돌하면서 숨은 실패경로가 나온다.

3. **지휘관이 바로 쓸 산출물이 있다.**
   긴 리포트가 아니라 1페이지 결심카드로 끝난다.
