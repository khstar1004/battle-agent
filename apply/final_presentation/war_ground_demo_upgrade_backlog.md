# WAR GROUND 시연 데모 고도화 리스트

자료 제출 2일 전 기준으로 심사 시연 임팩트와 안정성을 우선순위화한 목록입니다.

## P0 이번 반영 완료

- 3D 리허설 판단 브리핑: 현재 이벤트가 지형 관찰, 실패경로, 결심 조건으로 어떻게 이어지는지 3D 맵 위와 이벤트 카드 아래에서 바로 보이게 함.
- 3D URL 상태 동기화: `?stage=rehearsal&event=comm_gap`로 직접 열어도 Three.js 런타임이 해당 이벤트 카메라와 오버레이로 시작하게 함.
- 지휘결심카드 승인 게이트: 통신, 군수, 재판단, 후송 조건을 승인 게이트로 보여주고 조건 반영 여부와 잔여위험을 연결함.
- 최종 승인 흐름: AI 검토안, 참모 보완, 지휘관 승인, 단편명령 전파 순서를 human-in-the-loop 흐름으로 표시함.
- 심사 Q&A 큐: 각 단계별 예상 질문, 답변 프레임, 보여줄 위치를 상단 시연 큐에 추가함.
- 3D 캔버스 검증성: `preserveDrawingBuffer`를 켜서 headless 검증에서 캔버스 픽셀 샘플링이 가능하게 함.
- 5분 시연 러닝오더: 상단 콘솔에 발표 시간표, 단계별 멘트, 증명 포인트를 추가하고 현재 단계에 맞춰 하이라이트함.

## P1 제출 전 추가하면 좋은 항목

- 자동 진행 리허설 타이밍 보정: 자동 진행 모드에서 실제 발표 리듬과 맞는 3분/5분 프리셋 선택을 추가.
- 비교 시연 한 장 요약: 기존 단순 LLM 출력과 WAR GROUND 출력의 차이를 `근거 추적`, `실패경로`, `승인 게이트` 3축으로 압축.
- 모바일 시연 안전장치: 390px 화면에서 상단 시연 큐를 더 짧게 접는 모드 추가.

## P2 시간 여유가 있으면 좋은 항목

- 3D 카메라 프리셋 이름 정리: 군 용어 기준으로 `전체전장`, `음영구간`, `보급축`, `목표지역` 등으로 더 일관화.
- 산출물 다운로드 패키지: 결심카드 JSON 외에 브리핑 텍스트, 근거 목록, 스크린샷 참조를 하나의 제출 패킷으로 묶기.
- 발표자 리허설 로그: 자동 진행 중 각 클릭 시각과 예상 멘트를 저장해 발표 연습 체크리스트로 활용.

## 검증 기준

- `npm test` 전체 통과.
- Chrome headless에서 리허설/결심/모바일 화면 스크린샷 생성.
- 3D 캔버스 픽셀 샘플이 비어 있지 않음.
- 모바일 390px에서 새 3D 브리핑 스트립이 뷰포트 밖으로 넘치지 않음.
